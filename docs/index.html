Personal Project

Document Q&A

I built this project to make internal documentation easier to work with. It lets users upload documents, ask questions in plain language, and come back to saved conversations later instead of digging through long files or scattered notes.

GitHub Repository Demo Video

Project Focus

The main idea behind this project is to help teams get to the point faster when they are reading project material. A new developer joining a team usually has to go through requirement documents, design notes, feature writeups, and onboarding material before they understand what a project is really doing. I wanted a simpler way to ask direct questions, get a summary, and review the important context without manually searching through every document.

It is especially useful for internal knowledge-sharing workflows, and it can be extended to documentation sources such as Confluence pages so teams can quickly review project information, understand feature context, and make onboarding easier for new developers.

What It Does

  • Supports PDF, TXT, DOCX, and DOC uploads.
  • Extracts document text and answers questions in natural language.
  • Saves each user’s document sessions and question history.
  • Provides a three-panel workspace for chats, active Q&A, and history.
  • Helps with onboarding, project review, and understanding internal documentation.

Tech Stack

React Vite Flask SQLite Ollama PyPDF2 python-docx

Highlights

  • Built the full document Q&A flow end to end, from upload to saved conversation history.
  • Added authentication and user-specific storage so each person sees their own documents and chats.
  • Used chunked context retrieval to make answers more useful on longer documents.
  • Designed the interface as a workspace with saved chats, an active Q&A panel, and conversation history.

Architecture

  • Frontend: React handles sign-in, uploads, document selection, and the workspace UI.
  • Backend: Flask exposes the APIs for auth, uploads, saved history, and question answering.
  • Storage: SQLite stores users, uploaded documents, and saved chat history.
  • Model runtime: Ollama runs the local model used to answer document questions.
